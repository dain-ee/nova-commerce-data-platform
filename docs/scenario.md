# Nova 커머스 시나리오

## 회사 개요

Nova 커머스는 패션, 라이프스타일, 식품 카테고리를 운영하는 중대형 이커머스 플랫폼이다.

가정 규모:

- MAU: 500만
- 회원 수: 300만
- 상품 수: 100만
- 일 주문 수: 20만
- 주요 트래픽: 모바일 앱, 웹, 광고 캠페인, 쿠폰 프로모션

## 데이터 플랫폼팀의 문제

각 부서는 매일 데이터를 요청하지만, 데이터는 여러 서비스와 이벤트에 흩어져 있습니다.
예를 들어 주문 데이터는 Order Service, 결제 데이터는 Payment Service, 배송 데이터는 Shipment Service에서 발생합니다.
데이터플랫폼팀은 이 데이터를 수집, 정제, 집계하여 각 부서가 신뢰할 수 있는 지표를 조회할 수 있도록 만들어야 합니다.

## 데이터플랫폼팀의 목표

- 원천 이벤트를 Kafka로 수집
- Bronze / Silver / Gold 레이어 구성
- 부서별 핵심 KPI 제공
- 반복적인 데이터 요청을 자동화
- 장애 상황에서도 재처리 가능한 구조 제공
- 대시보드와 SQL 조회 환경 제공

## 주요 운영 시나리오

- 주문 이벤트 중복 발생
- 배송 이벤트 지연 도착
- 결제 성공 후 주문 상태 미변경
- 환불 이벤트 누락
- 주문 스키마에 coupon_id 컬럼 추가
- 잘못된 금액 데이터 유입
- DLQ 발생
- Backfill 필요
- Iceberg Small File 증가
- 배치 Job 실패
- Streaming Job 장애
